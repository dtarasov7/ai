1. Аутентификация и субъекты доступа (RBAC-001)
RBAC-001.01: Система должна поддерживать аутентификацию S3-запросов по схеме AWS Signature Version 4 (access key + HMAC на основе secret key) для всех защищённых операций.
RBAC-001.02: Неаутентифицированные запросы должны обрабатываться как запросы анонимного субъекта с применением тех же ACL/policy, что и к аутентифицированным.
RBAC-001.03 (опционально): Поддержка аутентификации через внешний IAM/Keystone-сервис с получением AWS-style ключей (EC2 credentials).
2. Мультитенантность и изоляция через placement/pools (RBAC-002)
RBAC-002.01: Система должна поддерживать мультитенантность через сущность tenant, обеспечивая изоляцию пользователей/бакетов с одинаковыми именами в разных tenant.
RBAC-002.02: Поддержка неявного tenant "по умолчанию" (из контекста учётных данных субъекта).
RBAC-002.03: Доступ к бакетам другого tenant через синтаксис tenant:bucket (path-style).
RBAC-002.04: Механизм placement targets с фиксированной привязкой к бакету при создании (неизменяема после создания).
RBAC-002.05: Каждая placement-цель сопоставляется с раздельными пулами:
Тип пула	Назначение
index_pool	Индексные данные бакета
data_pool (по storage class)	Данные объектов
data_extra_pool	Метаданные незавершённых multipart uploads
RBAC-002.06: Ограничение доступа к placement-целям через tags: создание бакета разрешено только при пересечении placement_tags пользователя/tenant с tags placement-цели.
