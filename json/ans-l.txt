**Краткий ответ:**  
**Да, в Grafana 10 можно построить такую таблицу, но с некоторыми ограничениями и нюансами.**

---

## Как это реализовать

### 1. **Импорт внешнего JSON с ожидаемыми версиями**

В Grafana 10 есть [Data source типа "JSON API"](https://grafana.com/grafana/plugins/marcusolsson-json-datasource/) (или похожие плагины), который позволяет подключить внешний JSON-файл или REST API как источник данных.

**Пример структуры JSON:**
```json
[
  {"service": "service1", "expected_version": "1.2.3"},
  {"service": "service2", "expected_version": "2.0.0"}
]
```

**Варианты:**
- Если файл лежит на сервере, можно поднять простой HTTP-сервер (например, nginx) и отдавать этот файл.
- Если это API — просто указываете URL.

---

### 2. **Получение реальной версии из Prometheus**

Допустим, у вас есть метрика:
```
myservice_info{service="service1", version="1.2.3"}
```
или аналогичная.

В Grafana делаете Prometheus-запрос:
```promql
myservice_info
```
или фильтруете по нужным лейблам.

---

### 3. **Объединение данных из двух источников**

**В Grafana 10** можно объединять данные из разных источников с помощью **Transformations** → **Join**.

**Пошагово:**
1. В панели Table добавьте два запроса:
   - **A:** JSON API (service, expected_version)
   - **B:** Prometheus (service, version)
2. В Transformations выберите **Join** по полю `service`.
3. В итоговой таблице будут:
   - service
   - expected_version (из JSON)
   - version (из Prometheus, можно переименовать в "real_version")

---

### 4. **Возможные ограничения и нюансы**

- **JSON API** — это внешний плагин, его нужно установить (через Grafana plugins).
- **JSON-файл** должен быть доступен по HTTP(S).
- **Join** работает только если значения поля `service` совпадают в обоих источниках.
- Если сервис есть в JSON, но нет в Prometheus — в поле "реальная версия" будет пусто (и наоборот).

| service   | expected_version | real_version |
|-----------|-----------------|-------------|
| service1  | 1.2.3           | 1.2.3       |

## **Вывод**

- **Да, это возможно**: используйте JSON API datasource + Prometheus + Transformations (Join).
- **Понадобится установка плагина** (JSON API).
- **Всё делается через UI Grafana, без скриптов и внешних ETL.**
