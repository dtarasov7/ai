Я подготовлю для вас полный пример Java приложения с OpenTelemetry agent, включая Docker-сборку и инструкции по запуску.Я подготовил для вас полный набор для тестирования OpenTelemetry Java agent:

## Что включено:

1. **Main.java** - Простое Java приложение, которое имитирует обработку заказов с несколькими методами. Это создаст хорошие traces с несколькими spans.

2. **pom.xml** - Maven конфигурация для сборки приложения.

3. **Dockerfile** - Многоступенчатый Docker файл, который:
   - Собирает Java приложение с помощью Maven
   - Скачивает последнюю версию OpenTelemetry Java Agent
   - Экспортирует оба файла в локальную папку

4. **Инструкции** - Подробное руководство по сборке и запуску.

## Основные возможности приложения:

- Циклически выполняет "обработку заказов"
- Вызывает несколько методов, создавая иерархию spans
- Случайно генерирует ошибки для тестирования error tracking
- Выводит логи для отслеживания работы

## Варианты запуска:

1. **С консольным выводом** - для быстрого тестирования, все traces будут в консоли
2. **С Jaeger** - для визуализации traces в веб-интерфейсе

Приложение готово к запуску и будет автоматически инструментировано OpenTelemetry agent'ом без изменения кода!
