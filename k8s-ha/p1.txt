Построй диаграмму катастрофоустойчивого (устойчивого к аварии ЦОД) k8s в формате plantuml. 
Имеется три ЦОД. сетевая задержка между ними не превышает 10ms.
DC1 и DC2 - для ingrеss, master и worker
DC3 только для master для обеспечения кворума
НЕ использовать GSLB.
Сеть между ЦОД'ами растянута по L2.
Persistent volumes обеспечиваются внешним по отношению к k8s кластером ceph.
Входной трафик на ingress балансируется с помощью 4-x серверов с haproxy (по два в каждом из ЦОД). 
Трафик на эти сервера поступает через один VIP (keepalived)
Обязательно проверь полученую схему на синтаксические ошибки.
