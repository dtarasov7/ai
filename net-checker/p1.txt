Необходимо написать роль на ansible
Имя роли net-checker.
Все переменные должны начинаться с префикса net_checker_
Цель роли - проверить сетевую связанность между хостами. Связанность проверяется либо 
по icmp (ping) либо по tcp (nc). Режим определяется переменной.
Порт для TCP определяется переменной.
Для проверки на хосты, с которых проверяется доступность загружается скрипт на bash, который формируется из темплейта. В процессе формирования 
в скрипте прописывается список хостов к которым проверяется связанность.
Результат выполнения Скрипта - метрики в формате Prometheus.
Возможны два варинат работы скрипта ( опеределяется переменой).

1-ый вариант 
Требуется,чтобы на серверах, где запускается скрипт был установлен node-exporter c модулем textcollector.
Наличие модуля проверяется по наличию на сервере определенной директории (задается через переменную). Если директории нет, то роль останавливается по ошибке
Если node-exporter установлен, то на сервере создается пользователь sys_net_checker и в его домашней директории загружается скрипт.
В crontab пользователя устанавливается запуск скрипт через установленое кол-во минут (определяется переменной).
Результат работы скрипта помещается в каталог для textcollector. На этот файл ставиться owner sys_node_exporter. Чтобы это можно было 
сделать от имени пользователя sys_net_checker, в  /etc/sudoers.d создается файл, разрешающий пользователю sys_net_checker 
выполнять sudo chown sys_node_exporter *
В роли должен быть код, который при использовании тега never удалял скрипт, пользователя , созданый скриптом файл и crontab пользователя и правила sudoer.

2-ой вариант
Скрипт загружается на сервер, выполняется , а результат выполнения скрипта запоминается на ansible хосте и сохраняется в 
файл с префиксом - именем целевого сервера.

Учитывая, что на одном сервер может одновременно работать несколько независимых проверок, надо чтобы для разных проверок различались 
имена скриптов, имена создаваетсых файлов с метриками и в самих меириках присутствовала метка, по которой можно было различать метрики 
разных проверок.

В метриках должны быть метки описывающие 
- сервер источник - сервер с которого проверяется доступность ( имя и IP) 
- сервер приемник - сервер к которму проверяется доступность ( имя и IP) 
- режим
- порт для TCP
Так же должны быть метки:
- env : содержимое переменной net_checker_env
- new_is : содержимое переменной net_checker_is
- new_instance : имя сервера источника
- new_job : содержимое переменной net_checker_job
