# Лабораторная работа 4. IAM-модель для shop-demo

## Цель

Построить понятную модель доступа для людей и автоматизации, не сводя всё к `cluster-admin`.

## Исходные материалы

- `shop-demo/README.md`
- глава 4 курса
- markmap `markmaps/chapter_04_access_management.mm`

## Ожидаемое время

75-105 минут

## Задание

1. Определите роли для:
   - разработчиков;
   - platform team;
   - аудиторов;
   - CI/CD.
2. Разделите доступ на:
   - namespace-level;
   - cluster-level;
   - аварийный доступ (break-glass).
3. Выберите, какая ролевая модель лучше подходит под ваш сценарий:
   - current;
   - experimental.
4. Опишите владельцев групп и процесс review прав.
5. Подготовьте policy для machine identities:
   - отдельные service accounts;
   - минимально необходимые права;
   - запрет смешивания с человеческим доступом.
6. Подготовьте YAML-примеры для выбранной модели:
   - `RoleBinding` или `AuthorizationRule` для группы разработки;
   - read-only доступ для аудиторов;
   - `ServiceAccount`, `Role` и `RoleBinding` для CI/CD в одном целевом namespace.

## Что нужно сдать

- IAM matrix `subject -> role -> scope -> owner`;
- решение по ролевой модели;
- RBAC-манифесты для людей и machine identities;
- схема review and revoke process;
- описание аварийного доступа (break-glass access).

## Критерии завершения

- люди и автоматизация разделены;
- у каждой группы есть владелец;
- исключения не подменяют стандартную модель доступа;
- есть понятный путь отзыва прав.
