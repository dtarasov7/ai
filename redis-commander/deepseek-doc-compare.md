# Redis Commander TUI - Описание функционала

## Основные возможности

### **1. Управление подключениями**
- Поддержка множества профилей подключений к Redis
- Автоматическая загрузка профилей из:
  - Обычного JSON файла (режим по умолчанию)
  - Зашифрованного файла (с использованием cryptography)
  - HashiCorp Vault (через API)
- Поддержка обоих режимов:
  - **Standalone Redis** (классический)
  - **Redis Cluster** (распределенный)
- Возможность переключения между базами данных (0-15 для standalone)
- SSL/TLS поддержка для безопасных подключений

### **2. Просмотр и навигация**
- **Иерархическое дерево**:
  - Серверы (с индикаторами подключения)
  - Базы данных (с подсчетом ключей)
- **Список ключей** с визуализацией:
  - Иконки типов ключей ([St], [Hs], [Li], и т.д.)
  - Подсветка выбранного ключа
  - Виртуализированный список для производительности
- **Фильтрация ключей**:
  - Поддержка wildcards (*, ?)
  - Поиск по подстроке

### **3. Работа с ключами**
- **Просмотр деталей**:
  - Тип ключа (string, hash, list, set, zset, stream)
  - TTL (время жизни)
  - Размер/длина
  - Для кластера: хэш-слот и узел
- **Операции**:
  - Добавление новых ключей (F3)
  - Редактирование существующих (F4)
  - Удаление ключей (F8)
  - Массовые операции с отметками:
    - Пробел - отметка/снятие отметки
    - Ctrl+A - отметить по шаблону
    - Ctrl+U - снять все отметки
    - Массовое удаление отмеченных

### **4. Консоль Redis (F2)**
- Полноценная Redis консоль с историей команд
- Поддержка всех Redis команд
- История команд (Page Up/Down для навигации)
- Форматированный вывод результатов
- Автоматическое обновление списка ключей после модифицирующих команд

### **5. Специальные возможности для кластера**
- Автоматическое определение кластера
- Отображение информации о кластере (состояние, узлы)
- Отображение хэш-слотов для ключей
- Правильная обработка кросс-слотовых операций
- Сканирование всех мастер-узлов для получения ключей

### **6. Визуальные особенности**
- Темная тема с цветовой дифференциацией
- Индикаторы прокрутки
- Статусная строка с информацией
- Диалоговые окна для всех операций
- Горячие клавиши для быстрого доступа

### **7. Безопасность и логирование**
- Аудит-лог всех операций
- Шифрование конфигурационных файлов
- Поддержка SSL сертификатов
- Изоляция паролей

## Сравнение с `redis-cli`

### **Преимущества Redis Commander TUI:**

| Аспект | Redis Commander TUI | redis-cli |
|--------|-------------------|-----------|
| **Визуальный интерфейс** | Полноценный TUI с деревом, списками, диалогами | Текстовый интерфейс, одна строка ввода |
| **Навигация по ключам** | Иерархическое дерево с фильтрацией, поиском, визуальными иконками | Только команды KEYS/SCAN, нет визуализации |
| **Работа с подключениями** | Множество профилей, быстрое переключение, SSL настройки | Одно подключение за раз, сложная настройка SSL |
| **Работа с типами данных** | Специализированные редакторы для каждого типа (hash, list, set и т.д.) | Общий SET/GET, сложный синтаксис для сложных типов |
| **Массовые операции** | Визуальная отметка ключей, массовое удаление, фильтры | Требует скриптов или сложных команд |
| **Кластер Redis** | Автоматическое определение, отображение узлов и слотов | Ручное указание узлов, сложная навигация |
| **История команд** | Визуальная история с форматированным выводом | Простая история в одной сессии |
| **Безопасность** | Шифрование конфигов, поддержка Vault, SSL с сертификатами | Пароли в командной строке (видимы в истории) |

### **Преимущества `redis-cli`:**
| Аспект | redis-cli | Redis Commander TUI |
|--------|-----------|-------------------|
| **Скорость выполнения** | Мгновенный запуск, минимальные задержки | Запуск 1-3 секунды, начальная загрузка ключей |
| **Потребление ресурсов** | Минимальное (терминальный клиент) | Выше (Python + urwid) |
| **Скриптование** | Легко интегрируется в скрипты, пайплайны | Только ручное взаимодействие |
| **Все команды Redis** | Полная поддержка, включая самые новые | Может отставать в поддержке новых команд |
| **Производительность** | Нет накладных расходов на UI | Визуализация требует сканирования и кеширования |
| **Портируемость** | Часть Redis, работает везде где есть Redis | Требует Python 3.6+ и библиотеки |

### **Удобство сравнение:**

**Для административных задач и разведки:**
- **Redis Commander TUI** значительно удобнее:
  - Быстрый обзор всей базы данных
  - Визуальная навигация по ключам
  - Понимание структуры данных
  - Массовые операции без написания скриптов

**Для разработки и отладки:**
- **Смешанное использование**:
  - TUI для исследования данных
  - `redis-cli` для быстрых команд и скриптов

**Для продакшн-мониторинга:**
- **Redis Commander TUI** лучше для:
  - Быстрой диагностики проблем
  - Понимания распределения данных в кластере
  - Визуальной оценки использования памяти

### **Скорость выполнения:**

**Операции просмотра:**
- **TUI**: Медленнее начальная загрузка (сканирование ключей), но после кеширования - быстрая навигация
- **redis-cli**: Мгновенная любая команда, но нет кеширования

**Модифицирующие операции:**
- Одинаковая скорость (оба используют тот же протокол Redis)

**Работа с большими объемами:**
- **redis-cli** лучше для пайплайнов и массовых операций
- **TUI** ограничен интерфейсом (максимум ~5000 ключей для отображения)

## Выводы

### **Используйте Redis Commander TUI когда:**
1. Нужен **визуальный обзор** структуры Redis
2. Работаете с **незнакомой** Redis базой
3. Требуется **быстрая навигация** по ключам
4. Работаете с **кластером Redis** и нужно понимать распределение
5. Нужны **массовые операции** без написания скриптов
6. Требуется **безопасное хранение** паролей (Vault/шифрование)

### **Используйте `redis-cli` когда:**
1. Нужна **максимальная скорость** выполнения
2. Работаете в **скриптах** или автоматизации
3. Используете **пайплайны** или сложные транзакции
4. Требуется **портability** без зависимостей
5. Работаете с **огромными** наборами данных (>100K ключей)
6. Используете **новейшие** команды Redis

### **Идеальный workflow:**
- **Для исследования и администрирования**: Redis Commander TUI
- **Для скриптов и автоматизации**: `redis-cli`
- **Для разработки**: Оба инструмента параллельно

Redis Commander TUI не заменяет `redis-cli`, а **дополняет** его, предоставляя визуальный интерфейс для задач, где это необходимо, сохраняя при этом возможность выполнять любые Redis команды через встроенную консоль.
