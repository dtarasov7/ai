Ниже готовый вариант: discovery сервисов + получение значения по `status` и `service`.

Допущение: Prometheus URL `http://127.0.0.1:9090`, label сервиса в метрике называется `service`.

`/etc/zabbix/scripts/ingress_services_discovery.sh`

```bash
#!/bin/bash
set -euo pipefail

PROM_URL="${PROM_URL:-http://127.0.0.1:9090}"

QUERY='count by (service) (nginx_ingress_controller_request)'

curl -sG "$PROM_URL/api/v1/query" \
  --data-urlencode "query=$QUERY" |
jq -c '
{
  data: [
    .data.result[]
    | select(.metric.service != null and .metric.service != "")
    | { "{#SERVICE}": .metric.service }
  ]
}
'
```

`/etc/zabbix/scripts/ingress_status_service.sh`

```bash
#!/bin/bash
set -euo pipefail

PROM_URL="${PROM_URL:-http://127.0.0.1:9090}"

STATUS="${1:?status required}"
SERVICE="${2:?service required}"

QUERY="sum(increase(nginx_ingress_controller_request{status=\"$STATUS\",service=\"$SERVICE\"}[1m]))"

VALUE="$(curl -sG "$PROM_URL/api/v1/query" \
  --data-urlencode "query=$QUERY" |
jq -r '.data.result[0].value[1] // 0')"

printf '%s\n' "$VALUE"
```

Права:

```bash
chmod +x /etc/zabbix/scripts/ingress_services_discovery.sh
chmod +x /etc/zabbix/scripts/ingress_status_service.sh
chown zabbix:zabbix /etc/zabbix/scripts/*.sh
```

Конфиг для `zabbix-agentd v1`, например:

`/etc/zabbix/zabbix_agentd.d/ingress_prometheus.conf`

```ini
UserParameter=ingress.services.discovery,/etc/zabbix/scripts/ingress_services_discovery.sh
UserParameter=ingress.status.service[*],/etc/zabbix/scripts/ingress_status_service.sh "$1" "$2"
```

Перезапуск агента:

```bash
systemctl restart zabbix-agent
```

Проверка:

```bash
zabbix_get -s 127.0.0.1 -k ingress.services.discovery

zabbix_get -s 127.0.0.1 -k 'ingress.status.service[500,my-service]'
zabbix_get -s 127.0.0.1 -k 'ingress.status.service[404,my-service]'
```

В Glabber/Zabbix-шаблоне:

```text
Discovery rule:
key: ingress.services.discovery

Item prototype:
name: Ingress HTTP {#SERVICE} status 500 requests per minute
key: ingress.status.service[500,{#SERVICE}]

Item prototype:
name: Ingress HTTP {#SERVICE} status 404 requests per minute
key: ingress.status.service[404,{#SERVICE}]
```

Если нужно разделять одинаковые `service` в разных namespace, лучше делать discovery по `namespace + service`.
